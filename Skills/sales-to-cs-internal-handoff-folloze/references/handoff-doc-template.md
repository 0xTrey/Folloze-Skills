# Handoff Doc Template

Use this as the default structure for the Google Doc handoff.

## Title block

- Customer name
- `Sales to Customer Success Internal Handoff`
- Date

## Recommended sections

### 1. Account Snapshot

Cover:

- signed date
- signed package / plan
- creator count
- collaborator count
- signed commercial reality

### 2. Current Signed Scope

List only the current signed scope.

Include:

- included platform elements
- excluded platform elements
- support model if known
- integrations included or explicitly excluded

### 3. Confirmed Users

Split into:

- champion
- confirmed admin users
- likely collaborators
- internal stakeholders

### 4. Priority Use Cases

Always include:

- first priority use case
- second-order use cases if supported
- why these matter to the customer

### 5. First Priority Campaign

Always include this section.

Describe:

- the first campaign or motion the customer should launch
- target audience
- why it is first
- what content or setup dependencies it needs

### 6. Tech Stack And Integrations

List:

- customer tech stack from deal notes
- integrations currently in scope
- integrations explicitly out of scope at launch
- future integration ideas only if supported

### 7. Launch Notes And Dependencies

Cover:

- launch timing
- subdomain need
- brand book / design setup
- privacy / cookie requirements
- procurement or technical blockers

### 8. Sales-Cycle Expansion Ideas

This must always be a separate section.

Include only clearly labeled future ideas such as:

- additional creator licenses
- integrations not in the signed package
- larger rollout across teams
- adjacent use cases
- upsell / cross-sell opportunities

### 9. Open Questions

Use this when important information is still missing.

### 10. Audit Log

Always end with:

- sources referenced
- linked docs opened
- emails / meetings used
- assumptions made
- issues encountered
- artifacts created

## Tone

- internal only
- operational
- concise
- no confidence labels unless the user asks
- best-supported statements only
