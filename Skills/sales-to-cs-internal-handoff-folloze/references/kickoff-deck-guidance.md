# Kickoff Deck Guidance

Use this guidance when updating or creating the onboarding kickoff presentation.

## Preferred order

1. Update the customer's existing kickoff deck if the user provided a link.
2. Otherwise search the customer Drive folder for an existing kickoff deck.
3. Otherwise locate the standard template.
4. If no template can be found, create a clean internal kickoff deck and log that assumption.

## Minimum sections

### 1. Cover

- customer name
- Folloze onboarding kickoff
- date

### 2. Team

- customer champion
- confirmed admins
- known stakeholders
- Folloze internal owners if available

### 3. Signed Scope

- current signed package
- creator count
- collaborator count
- what is included
- what is excluded

### 4. Priority Use Cases

- first priority use case
- secondary use cases if supported by evidence

### 5. First Priority Campaign

- what the first campaign should be
- target audience
- why it is first
- what content or setup is required

### 6. Tech Stack And Setup

- relevant martech stack
- integrations in launch scope
- integrations explicitly out of scope
- subdomain, brand book, privacy requirements

### 7. Launch Path

- target launch timing
- dependencies
- blockers
- immediate next steps

### 8. Expansion Ideas

Keep this clearly separated from signed scope.

## Style rules

- internal and operational
- concise
- no speculative claims presented as fact
- signed scope first
- expansion ideas last

## If the deck is customer-specific already

- preserve the existing structure where it is usable
- update facts, scope, and use cases
- remove stale assumptions from earlier sales-cycle versions
