# Common Gating Models

## Governing Principle

Give the buyer useful value before asking for conversion. Do not hide all substance behind a form, and do not give away the entire gated asset if the user has specified that the full asset should remain gated. The ungated portion should educate, qualify, and build intent; the gated action should feel like a natural next step.

Never invent a live asset URL, form destination, registration link, or downloadable file. If the final asset or destination is missing, mark it as pending and surface that caveat before saving or publishing.

## Model 1: Ungated Summary / Gated Full Asset

Use when:
The user has a guide, report, ebook, whitepaper, or long-form content asset and wants visitors to understand the value before downloading.

Structure:
- Hero with the core promise.
- Ungated executive summary or key takeaways.
- Preview of 3-5 themes, chapters, or lessons.
- Optional interactive checklist, tabs, or chapter selector.
- CTA to download the full asset.

Ungated value:
- Key problem.
- Main takeaways.
- Who the asset is for.
- What the buyer will learn.

Gated value:
- Full guide, full report, full PDF, complete framework, templates, worksheets, or detailed recommendations.

Avoid:
- Reproducing the full asset ungated.
- Making the ungated page so vague that the form feels premature.
- Linking to a fake or placeholder download.

Example CTA:
"Download the full guide"

## Model 2: Checklist-Gated Guide

Use when:
The user wants buyers to complete a short checklist before downloading a guide, playbook, or framework.

Structure:
- Hero with the outcome.
- Short ungated checklist with 5-8 items.
- Interactive checkboxes or progress state.
- Lightweight interpretation of the visitor's selections.
- CTA to get the complete guide.

Ungated value:
- Self-assessment.
- Quick readiness snapshot.
- Priority areas surfaced by the checklist.

Gated value:
- Full guide, implementation steps, detailed checklist, worksheet, or recommended action plan.

Avoid:
- Collecting sensitive information in the checklist.
- Making the checklist feel like a quiz with unsupported scoring.
- Claiming a precise diagnosis unless the scoring model is approved.

Example CTA:
"Get the complete checklist"

## Model 3: Assessment-Gated Report

Use when:
The experience should help the buyer evaluate maturity, readiness, performance, or fit before requesting a full report.

Structure:
- Hero with the assessment promise.
- 4-7 question assessment.
- Simple result category or readiness band.
- Preview of what the result means.
- CTA to receive the full report or recommended plan.

Ungated value:
- Assessment questions.
- High-level result.
- A few recommended focus areas.

Gated value:
- Detailed report, personalized recommendations, benchmark-style explanation, or next-step plan.

Avoid:
- Fake benchmarks.
- Unsupported maturity scores.
- Overly personal or sensitive questions.
- Claiming the report is personalized if it is only a generic follow-up.

Example CTA:
"Get my full report"

## Model 4: Webinar Companion

Use when:
The source content is a webinar, event recording, live session, or webinar concept.

Structure:
- Hero with the webinar topic and audience.
- Why-watch or why-attend section.
- Key themes or agenda preview.
- Speaker/resource area if approved.
- Related resources or prep checklist.
- CTA to register, watch, or get the recap.

Ungated value:
- Topic framing.
- Agenda or key questions.
- Speaker/resource preview.
- Useful takeaway before registration.

Gated value:
- Live registration, recording access, slides, recap, or post-event resource pack.

Avoid:
- Inventing speaker names, dates, times, recordings, or registration links.
- Adding countdowns or calendar links unless event details are confirmed.
- Treating a webinar companion like a static event flyer.

Example CTA:
"Register for the webinar" or "Watch the recording"

## Model 5: Chapter Preview

Use when:
The asset has clear sections, chapters, lessons, or modules and the user wants buyers to explore before converting.

Structure:
- Hero with the asset promise.
- Chapter selector, tabs, cards, or accordion.
- Short preview for each chapter.
- Recommended path based on role, priority, or stage if provided.
- CTA to unlock the full asset.

Ungated value:
- Chapter summaries.
- Key questions each chapter answers.
- Preview snippets or practical takeaways.

Gated value:
- Full chapters, complete asset, templates, examples, or implementation details.

Avoid:
- Publishing full chapters if the asset should remain gated.
- Creating chapter titles that imply unsupported content.
- Making every chapter preview identical or generic.

Example CTA:
"Unlock the full guide"

## Choosing The Right Model

Use ungated summary / gated full asset when the buyer needs a quick overview before committing.

Use checklist-gated guide when the buyer should self-identify gaps or priorities.

Use assessment-gated report when the campaign wants qualification, readiness, or maturity framing.

Use webinar companion when the asset is time-based, speaker-led, or event-driven.

Use chapter preview when the asset is structured and the buyer benefits from browsing before downloading.

## Final QA

Before final response or publish, verify:
- The gating model matches the user's requested conversion action.
- The buyer gets real value before the gate.
- The full gated asset is not accidentally reproduced ungated.
- CTA labels match the actual action.
- No fake form destination, download URL, registration link, recording, or asset file is invented.
- Pending asset links are clearly marked as pending.
- The final response explains what is ungated, what is gated, and what is still pending.
