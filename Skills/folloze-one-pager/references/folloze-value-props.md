# Folloze Value Props

Use this reference to map account goals and objections into Folloze one-pager copy.

Load `folloze-brand-kit` first when it is available. This file is the lightweight fallback menu and one-pager-specific pattern library, not the sole source of truth.

## Core Narrative

Folloze helps GTM teams build, personalize, and prove digital buyer experiences:

- Scale: launch on-brand campaign pages, microsites, deal rooms, partner hubs, and content destinations faster without adding headcount.
- Engage: personalize journeys by account, segment, persona, source, role, stage, and known or unknown identity.
- Impact: capture first-party engagement signals, content consumption, source paths, buyer heat, and sales-ready insight.
- Govern: support enterprise-ready programs with brand control, security review, permissions, scalability, and repeatable launch operations.

## Value Prop Menu

| Account need or objection | Folloze framing | One-pager card idea |
|---|---|---|
| Campaigns take too long to launch | Reuse a governed workspace to move from prompt, brief, or assets into campaign pages quickly | Faster campaign creation without rebuilding each destination |
| Buyer journeys are too generic | Personalize by account, persona, source, and stage so each visitor sees a relevant path | More relevant engagement from account and persona personalization |
| Known and unknown buyers arrive from multiple channels | Connect email, LinkedIn, website, forms, cookies, and known systems into a clearer identity path | Known buyers, partial signals, and fewer broken journeys |
| Sales needs proof, not vanity clicks | Show what buyers consumed, time spent, account activity, and what should move to sales | First-party intent and sales-ready insight |
| Agency, partner, or client programs need separation | Support internal programs, client tokens, managed service paths, and white-label/domain needs | Agency and client program routes |
| Security or procurement will slow momentum | Package enterprise readiness, SOC 2 Type 2 availability, governance, and global launch experience | Vendor review details ready for approval |
| Teams need a repeatable launch model | Standardize the build, activation, and reporting path so each program starts from a proven system | Launch motion in action |
| Leadership asks how this helps pipeline | Tie engagement depth, content consumption, and account heat back to pipeline-ready follow-up | Prove every ABM journey |

## Numeric Claim Discipline

Do not introduce new metrics casually.

Allowed sources for numeric claims:

- user-approved prior one-pagers
- current Folloze sales material
- verified customer proof or public case studies
- explicit user-provided assumptions for that account

If no approved numeric claim exists, use non-numeric value language. For example, use "faster campaign creation" instead of "5x faster campaign creation."

## Default Value Cards

The Younion-style template can start with these cards when the user has approved them for the motion:

- `5x` faster campaign creation without rebuilding each landing page
- `4-5x` more relevant buyer engagement from account and persona personalization
- `3 hours` typical build window once assets and brand are ready

Replace these when the account conversation points to stronger goals, objections, or proof needs.

## Copy Rules

- Write from Folloze's point of view, but make the page about the account's next move.
- Prefer the current category language from the Brand Kit: AI orchestration platform for B2B go-to-market teams.
- Use ABX as a capability when relevant, not as the lead category.
- Do not say "call notes", "meeting recap", "demo", "template", or "AI-generated" in buyer-facing copy.
- Name the account's business motion when it is safe: ABM launch, website personalization, partner programs, customer expansion, event follow-up, sales enablement, or deal-room acceleration.
- Pair every objection with a Folloze support path. The page should show how Folloze helps, not simply document concerns.
