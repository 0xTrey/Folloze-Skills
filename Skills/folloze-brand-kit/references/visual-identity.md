# Visual Identity

Use this file for Folloze logo, color, and visual usage guidance in generated assets.

## Logo Assets

Bundled logo files:

| File | Use |
|---|---|
| `assets/logos/folloze-logo-primary.svg` | Preferred vector logo for light backgrounds |
| `assets/logos/folloze-logo-primary.png` | High-resolution logo for light backgrounds when SVG is unsuitable |
| `assets/logos/folloze-logo-secondary.png` | Alternate logo treatment from the approved kit |
| `assets/logos/folloze-logo-white.png` | White logo for dark backgrounds |
| `assets/logos/folloze-symbol-primary.png` | Symbol-only mark for light backgrounds |
| `assets/logos/folloze-symbol-secondary.png` | Alternate symbol-only mark |
| `assets/logos/folloze-symbol-white.png` | White symbol-only mark for dark backgrounds |

Default choice:

- Use `folloze-logo-primary.svg` on light or pale backgrounds.
- Use `folloze-logo-white.png` on dark navy, blue, or image backgrounds.
- Use symbol-only marks only when space is constrained or when the Folloze wordmark is already visible nearby.

## Logo Rules

- Use the real logo asset. Do not redraw it in HTML or CSS.
- Do not stretch, skew, crop, rotate, outline, shadow, recolor, or apply CSS filters to the logo.
- Keep clear space around the logo.
- Keep the logo readable at the rendered size.
- Inspect the actual render before final export.
- For co-branded assets, use a small separator such as `x` between Folloze and the account/company mark when the design supports it.

## Visual Direction

Folloze-branded assets should feel:

- modern
- crisp
- enterprise-ready
- bright without being playful
- motion-oriented
- easy to scan
- confident but not hype-heavy

Recommended patterns:

- large clear outcome headline
- restrained gradients using light blue, white, and pink accents
- dark navy panels for orchestration, proof, or system views
- compact value cards with clear metrics or claims
- pill labels and chips for capability groups
- plenty of spacing around logo lockups

Avoid:

- one-note purple or dark-blue palettes
- decorative blobs or generic abstract backgrounds
- oversized marketing hero layouts when the asset is a working follow-up page
- cards inside cards
- unreadable small logo treatments
- using the Folloze symbol where the full wordmark is needed

## PDF And HTML Checks

Before sending an HTML or PDF asset:

- verify the logo renders from the bundled or copied asset path
- verify dark-background logo contrast
- verify print/PDF export does not introduce bands or missing backgrounds
- verify the PDF is one page when the format is intended as a one-page leave-behind
- verify all text remains readable at email attachment scale
