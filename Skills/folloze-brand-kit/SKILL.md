# Folloze Brand Kit

Use this skill as the portable Folloze brand and messaging source for Codex work. It is intentionally reusable across other Folloze skills.

This package contains public-facing, generally reusable Folloze brand material. It excludes restricted customer-specific materials, pricing, contracts, order forms, deal notes, and private account drafts.

## How To Use

For most work:

1. Read `references/source-index.md` to choose the minimum relevant references.
2. Use `references/brand-foundation.md` for category, positioning, audience, and narrative rules.
3. Use `references/messaging-library.md` for value props, pillars, approved phrases, and page/module copy options.
4. Use `references/claims-and-proof.md` before making any numeric or customer-proof claim.
5. Use `references/voice-and-style.md` for tone, words to use, and words to avoid.
6. Use `references/visual-identity.md` and `assets/logos/` for logo and color usage.

Do not load every reference by default. Use only what the task needs.

## Source Priority

Use current user-provided direction first when it is clearly intended for the asset being created. Then use this bundled brand kit. If the user provides a newer approved Folloze deck, website copy, or messaging guide that conflicts with this package, follow the newer source and note the conflict in the working summary.

For account-specific assets, pair this brand kit with the account research and call-note evidence. Do not let generic Folloze messaging override what the account actually cares about.

## Reuse By Other Skills

Other Folloze skills should reference this package instead of copying brand rules into their own instructions. Good downstream usage:

- `folloze-one-pager`: source approved value props, claims, proof, logos, and voice rules.
- `Folloze-MCP-Demo-Builder`: use logo assets and public Folloze positioning only when the user explicitly wants a Folloze-branded experience.
- `folloze-sales-doc`: use claim, proof, and voice references for customer-facing Folloze documents.
- outbound or follow-up skills: use the message pillars and proof library, then adapt to the account context.

If a downstream skill needs a new reusable message pattern, add it here once it is proven reusable and public-safe.

## Boundaries

- Do not add customer-specific notes, commercials, contract language, pricing, private deal strategy, or restricted customer proof to this skill.
- Do not claim full autonomy, AI replacing marketers, predictive planning, or future roadmap features.
- Do not lead with "ABX platform" as the primary category. ABX is a capability within the Folloze orchestration story.
- Do not use retired category language such as "Buyer Experience Platform" unless discussing historical context.
- Keep messaging grounded in current product reality and approved proof.

## Quick Brand Anchor

Folloze is an AI orchestration platform for B2B go-to-market teams. It helps revenue teams launch, optimize, and prove autonomous campaigns from prompt to pipeline.

The three core pillars are:

- Scale: do more with less.
- Engage: campaigns that improve themselves.
- Impact: revenue visibility.
